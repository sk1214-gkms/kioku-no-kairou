# 3Dで背景を作る 実作業マニュアル（初心者向け・見ながら手を動かす用）

**この1冊で「何を・どの順で・どう作るか」が分かる**ようにした説明書。Blender未経験・MMDは触ったことがある人向け。
アート方針は **3Dレンダリング（部屋を3Dで組み、カメラを4方向に回して静止画にする＝Myst型）で確定**。取り込みは **MMDステージ→Blender** を主線、見つからなければ **CC0素材** で代用。

- 各部屋に「何を置くか（情景・焦点）」の詳しい描写 → [画像の理想像](画像イメージ_自然言語.md)
- 作る枚数の全リスト → [アセット完全チェックリスト](アセット完全チェックリスト.md)
- 取り込みの技術詳細（プラグイン等） → [Blender制作手順_MMD取り込み](Blender制作手順_MMD取り込み.md)
- この文書＝**上記を「作業手順」に落とした主線**。まずこれを上から順にやる。

## 0. まず全体像（ゴールと道具）

### 作るもの（合計）
| 種別 | 枚数 | 保存先 | 優先 |
|---|---|---|---|
| 背景（通常）13室×4方向 | **52枚** | `assets/images/rooms/r1_north.png` … | ★最優先 |
| 背景（状態差分）R3/R8/R10/R9 各4方向 | **16枚** | `..._lit/_dark/_on/_open.png` | ★必須 |
| アイテムアイコン（透過） | **15個** | `assets/images/items/xxx.png` | ★必須 |
| 結末キービジュアル | 〜7枚 | 任意 | 後回しOK |
| タイトル背景 | 1枚 | 任意 | 後回しOK |

### 完成の定義（1枚ごと）
- 縦長 **1080×1920**（9:16）のPNG。
- 「調べる家具＝焦点」が画面の中央〜やや下に、はっきり写っている。
- 暗くてもいいが**焦点は視認できる**明るさ。
- 正しいファイル名で正しいフォルダに保存されている。

### 使う道具（すべて無料）
- **Blender 4.2 LTS**（3Dソフト）
- **MMD Tools**（Blenderに.pmxを読み込むアドオン）
- 素材：**商用OKのMMD室内ステージ** か **CC0素材**（Poly Haven 等）
- 仕上げ（任意）：Photoshop / GIMP / Krita（色をそろえる）

### 進め方の鉄則（初心者が挫折しないコツ）
1. **いきなり13室やらない。** まず「共通の設定」を1回作り、次に**R1だけ**を最後まで通す。
2. **部屋は1個作れば4枚撮れる**（カメラを回すだけ）。作り込みは1室1回。
3. **同じ"館の骨組み"を使い回す**。壁・床・扉は共通、家具だけ差し替える＝全室が「同じ館」に揃う＆速い。

## 1. 準備（最初の30分・1回だけ）

1. **Blender 4.2 LTS を入れる**（公式サイト・無料）。
2. **MMD Tools を入れる**：`Edit > Preferences > Add-ons` → 右上 **▼ > Install from Disk…** → ダウンロードした zip を選ぶ → 一覧の「mmd_tools」に**チェック**。
   - 確認：3D画面で **N キー** → 右に「MMD」タブが出ればOK。
   - 詳しい導入は [Blender制作手順_MMD取り込み §1](Blender制作手順_MMD取り込み.md)。
3. **素材を1つ用意**（下の「素材の選び方」参照）。まずは室内が1つあれば始められる。
4. 保存用に、ゲームの `assets/images/rooms/` と `assets/images/items/` の場所を確認しておく。

### 素材の選び方（商用配信するので重要）
本作は Google Play / App Store で**有料配信**＝商用。素材のライセンスは必ず確認。
- **MMDステージ**：配布元の `readme.txt` で**「商用利用可」かつ「改変可」**のものだけ使う（商用不可が多いので注意）。
- **不安なら CC0**（著作権フリー・完全自由）が安全：**Poly Haven**（家具・テクスチャ・HDRI）、**ambientCG**、Blender内蔵の **BlenderKit**（無料枠）。
- 迷ったら：**箱型の部屋を自分で作り（後述）＋Poly HavenのCC0家具を置く**のが、ライセンス的に一番安全で初心者にも作りやすい。

## 2. 「館テンプレート」を1回だけ作る（超重要・時短の要）

全部屋で使い回す土台ファイル。**これを作れば、以降は家具を替えて撮るだけ**。

### 2-1. 出力を縦長9:16に
- 右側の **Output Properties（プリンタ型アイコン）** → Resolution **X=1080 / Y=1920**。

### 2-2. レンダラを軽い方に
- **Render Properties（背面カメラ型アイコン）** → Render Engine = **EEVEE**（軽い・初心者向け。重いCyclesは後で選べる）。
- 同じ場所の **Color Management > View Transform** = **AgX**（映画的な階調。Photoshopで色を作り込むなら Standard でも可）。

### 2-3. 部屋の"箱"を作る（MMDステージを使うなら飛ばしてOK）
1. 最初からある立方体を選び **Tab キー**で編集モード → 大きさを部屋サイズに（例：幅6m×奥6m×高3m）。面を内向きにして「箱の中」を部屋にする（初心者は "Add > Mesh > Plane" を6枚並べて壁・床・天井にしてもよい）。
2. 床・壁に Poly Haven のCC0テクスチャを貼ると一気に"それらしく"なる。

### 2-4. カメラ装置（4方向を回すだけにする仕掛け）
MMDのカメラ回転と同じ発想。**中心に軸、その軸を90°ずつ回す**。
1. `Add > Empty > Plain Axes` を**部屋の中心・高さ 1.5m** に置く（＝目の高さ）。
2. `Add > Camera` を追加。カメラを選び、続けて Empty を **Shiftクリック**で両方選択 → **Ctrl+P > Object**（カメラをEmptyの子にする）。
3. カメラを**水平に前へ向ける**：カメラの **Rotation X = 90°**、位置は中心付近。
4. カメラの画角：**Object Data Properties（カメラ型）> Focal Length = 28mm**（室内が広く写る）。Sensor Fit = Auto。
5. 確認：**テンキー0** でカメラ視点。`Ctrl+テンキー0` で「選択カメラを現在の視点に」。

```
   （上から見た図）      北
                         ↑
              西 ← [Empty＋子カメラ] → 東     ← Empty の Z回転 0/90/180/270 で向きが変わる
                         ↓
                         南
```

### 2-5. 雰囲気の照明（館のDNA＝暗い・ティール＆赤）
1. **World Properties（地球儀アイコン）** → Color をほぼ黒（明度 **0.01**）。
2. **主光**：`Add > Light > Area` を1つ、部屋の片側・少し上に。**Power は 150W くらいから**始め、**目で見て"薄暗いが焦点は見える"**まで調整。少し青緑(ティール)寄りの色に。
3. **赤の差し色**：`Add > Light > Point` を1つ、赤・**Power 20W くらい**、弱く。血の記憶の差し色。

### 2-6. テンプレートとして保存
- `File > Save As…` → **`館テンプレート.blend`**。以降の各部屋はこれを複製して作る。

## 3. 最初の1部屋を最後まで（R1・題材つき）

**R1 白い部屋**を例に、1室を完成させる流れ。ここを1回通せば、残りは繰り返しです。

### R1 に置くもの（4方向の焦点／詳しくは[画像の理想像 R1](画像イメージ_自然言語.md)）
- **北＝洗面台**（古い白磁・下扉が半開き）
- **東＝ベッド**（鉄フレーム・シーツに暗い染み）
- **南＝扉**（施錠された白い扉・鍵穴）
- **西＝鉄格子の窓**（外は濃霧）

### 手順
1. `館テンプレート.blend` を複製 → `r1.blend` にリネームして開く。
2. **家具を置く**：洗面台・ベッド・扉・窓を、それぞれの壁際に配置。
   - MMDステージなら該当小物を、CC0なら Poly Haven / BlenderKit から取得して配置。
   - **細部より"シルエットで何か分かる"を優先**（プレースホルダより良ければ合格）。
3. **1方向ずつレンダリング**：
   - Empty の **Z回転 = 0°**（北を向く）→ **F12** でレンダリング → 出た窓で `Image > Save As…`（Alt+S）→ **`assets/images/rooms/r1_north.png`**。
   - Z回転 **90°** → `r1_east.png` ／ **180°** → `r1_south.png` ／ **270°** → `r1_west.png`。
   - ※どの回転がどの方角になるかは1回撮って確認し、以降は固定でOK。
4. **ゲームで確認**：`r1_*.png` を4枚入れて起動→R1が自分の絵になっていれば成功。
5. **暗すぎ/明るすぎ**はテンプレートの主光Powerを直し、4枚撮り直す。

> ✅ ここまで来たら「1室の作り方」を体得済み。あとは同じことを12回。

## 4. 状態差分（16枚）＝ライトを変えて撮り直すだけ

対象は **R3・R8・R10・R9**。同じ部屋で**光や物の状態を変えて再レンダリング**し、`_サフィックス` を付けて保存。
| 差分 | やること | 保存名の例 |
|---|---|---|
| **R3 `_lit`** | 青いブラックライトを足し、**北の壁に光る数字（○△□）**を出す（発光板 or Emissive） | `r3_north_lit.png` ほか4方向 |
| **R8 `_dark`** | 主光をほぼ消す。**東の鏡に人影＋赤い滴り**を写す | `r8_east_dark.png` ほか4方向 |
| **R10 `_on`** | **北の記録端末／東の補助モニタ**を Emission で発光（シアン）。部屋全体も少し明るく | `r10_north_on.png` `r10_east_on.png` ほか |
| **R9 `_open`** | **西の戸棚を開いた状態**にする（扉を回転・中は空） | `r9_west_open.png` ほか4方向 |

- コツ：通常版(`r3_north.png`等)と差分版(`r3_north_lit.png`等)は**同じカメラ**で撮る。ライト/物だけ変えてもう一度 F12 → 別名保存。
- **通常版には"出現物"を描かない**（例：R10通常はモニタ消灯、`_on`だけ点灯）。

## 5. アイテムアイコン15個（透過PNG）

小物を**背景透過**で撮る。2通り：
- **Blenderで撮る**：`Render Properties > Film > Transparent` に✓ → 小物1個を中央に置き、正面から撮る → PNG（透過）で `assets/images/items/item_r1_key.png` 等へ。名前は[チェックリストC](アセット完全チェックリスト.md#c-アイテムアイコン透過png-15種-必須)のとおり。
- **面倒なら後回し**：現状のプレースホルダ（コード生成の透過アイコン）でゲームは動くので、背景52＋差分16を先に完成させ、アイテムは最後でよい。

## 6. 色を全部そろえる（最後の仕上げ）

「同じ館」に見せる総仕上げ。どちらか一方でOK：
- **Blenderのコンポジット**：`Compositing` タブ → `Use Nodes` → **Color Balance** ノードで影をティール、中間をセピアに。**全部屋で同じ設定**を使う。
- **Photoshop/GIMPで一括**：全PNGに同じ調整（トーンカーブ/カラーバランス）をバッチ適用。

## 7. 13室を回す段取り（量産フェーズ）

1. `館テンプレート.blend` を複製 → `r2.blend`, `r3.blend`… と作る。
2. 各室：**家具を差し替え**（壁・床・扉・カメラ・ライトは使い回し）→ 4方向レンダリング → 保存 → 差分があれば撮り増し。
3. 各室に置く焦点は [画像の理想像](画像イメージ_自然言語.md) と [チェックリストA](アセット完全チェックリスト.md#a-背景通常-52枚-必須) を見る。
4. 進捗は下の付録チェックリストで管理。**1室あたり慣れれば30〜60分**が目安。

## 8. つまずき集（初心者向け）

- **真っ黒で何も映らない** → World が暗すぎ or ライト不足。まず主光Powerを上げて位置を確認。
- **カメラが床や天井を向く** → カメラの Rotation X=90° を忘れている。
- **横長になる** → Output の Resolution が 1080×1920 か確認。
- **重い/落ちる** → Eeveeを使う。確認中は解像度を半分にして、本番だけ1080×1920。
- **材質が真っピンク** → テクスチャのリンク切れ。貼り直す。
- **MMDステージが商用可か不明** → 使わない。CC0素材に切替。
- **ゲームに反映されない** → 保存パス/ファイル名が違う。`assets/images/rooms/r1_north.png` の形か確認（未配置だと暗いプレースホルダのまま）。

### ミニ用語辞典
- **メッシュ**＝3Dの形（壁・家具）。**マテリアル**＝表面の色/質感。**ライト**＝光源。
- **Empty**＝目印用の空オブジェクト（今回はカメラの回転軸）。
- **レンダリング(F12)**＝3Dを1枚の絵に焼く。**Eevee**＝速い描画方式。
- **Emission**＝自分で光る材質（モニタ画面など）。**透過(Transparent)**＝背景を抜く。
