# 背景の作り方：MMDステージ → Blender 取り込み → 背景レンダリング

**対象**：Blender未経験・MMDは触ったことがある人。
**目的**：3Dで部屋を1個用意し、カメラを4方向に回して**同じ館の背景を量産**する（一貫性が構造的に保証される）。
**この作品の出力先**：`assets/images/rooms/<id>_<dir>.png`（例 `r1_north.png`）。同じパスに置けば即差し替わる（未配置でも暗色フォールバックで落ちない）。

> 全体像は [画像制作の手法比較（3D＝Myst型の王道）](画像イメージ_自然言語.md)、絵の理想像は [画像の理想像](画像イメージ_自然言語.md)、色統一の考え方は [Firefly §6](AdobeFirefly_生成手順.md)。

---

## ⚠️ 最重要：ライセンス（広告収益＝商用のため必須）
本作は広告収益前提＝**商用利用**。MMDステージ/モデルは**配布元ごとに規約が全く違い、「商用不可」「改変不可」も多い**。
- 使う前に必ず同梱の `readme.txt` / 配布ページの規約を確認し、**「商用利用可」かつ「改変可」**のものだけ使う。
- 不明・不可のものは使わない。→ その場合は**CC0素材でBlender内に作る**方に切替（[代替ルート](#代替ルートcc0素材で作る商用が確実)参照）。Poly Haven / ambientCG は **CC0（完全自由）**で安全。
- MMDステージを土台に**大きく作り替える**場合でも、元の改変・再配布可否の範囲を超えないこと。

---

## 0. 必要なもの
- **Blender 4.2 LTS**（無料・長期サポート版。以降のメニュー名はこれ基準）
- **MMD Tools アドオン**（MMDの `.pmx` を読み込むプラグイン）
- 商用可の **MMDステージ**（`.pmx`）1つ（まずは室内が1つあれば足りる）
- （任意）**PMXエディタ**：MMD側でステージを整える／`.x`を`.pmx`化する等。使い慣れていれば活用

---

## 1. Blender と MMD Tools を入れる
1. Blender 4.2 LTS を公式サイトから入れる（オフライン可）。
2. **MMD Tools** を入れる：
   - `Edit > Preferences > Add-ons` を開く。
   - 右上の **▼（ドロップダウン）> Install from Disk…** で、ダウンロードした MMD Tools の zip を選ぶ。
     （Blender 4.2 では「Get Extensions」タブからオンライン取得も可。社内ネット不通なら zip 手動が確実）
   - 一覧の **「mmd_tools」にチェック**を入れて有効化。
3. 確認：3Dビューポートで **N キー** を押すと右にサイドバーが出て、**「MMD」タブ**があればOK。

---

## 2. MMDステージを読み込む
1. `File > Import > MikuMikuDance Model (.pmd, .pmx)` を選ぶ（MMD Tools有効化で出現）。
2. 商用可のステージ `.pmx` を選んで取り込む。
3. 見た目が暗い/材質が変なときは、まずはそのまま進めてOK（後でライトと色で整える）。
   - MMDの材質はトゥーン寄り。写実に寄せたい場合は各マテリアルを **Principled BSDF** に置き換えるが、**最初は既定のままで一度レンダリングして流れを掴む**のが吉。
- `.x` 形式のアクセサリしか無いステージは、PMXエディタ等で `.pmx` に変換してから取り込む。

> 💡 MMDでの感覚そのまま：「ステージを読み込む」→「カメラを置く」→「出力」。Blenderでも同じ順序です。

---

## 3. 出力サイズを縦長9:16にする
`Output Properties`（プリンタ型アイコン）で：
- **Resolution X = 1080 / Y = 1920**（縦長。ゲームの設計面 360×640 と同比率）
- Frame Rate等はレンダリング画像なので不問。

---

## 4. カメラを部屋の中心に置き、4方向に回す（キモ）
**「中心に軸を置き、その軸を90°ずつ回す」**＝MMDのターンテーブルと同じ発想。

1. `Add > Empty > Plain Axes` を部屋の中心・目の高さに置く（位置例：室中央、**Z ≈ 1.5m**）。
2. `Add > Camera` でカメラを追加。カメラを選び、続けて Empty を **Shift+クリック**して両方選択 → `Ctrl+P > Object`（カメラをEmptyの子にする＝親子付け）。
3. カメラを **水平に前を向かせる**：カメラの回転 `Rotation X = 90°`、位置はEmpty近く（中心）に。
4. **Empty の Z回転**を 0° / 90° / 180° / 270° に変えるだけで、**北→東→南→西の4方向**になる。
   - `Ctrl+Numpad0` で「選択カメラを現在の視点に」。**Numpad0** でカメラ視点確認。
5. 画角：室内は少し広角が見やすい。カメラの **Focal Length ≈ 24〜35mm**（`Object Data Properties`＝カメラ型アイコン）。**Sensor Fit = Auto**（縦解像度に自動対応）。

> 焦点（洗面台・鏡など）が画面中央〜やや下に来るよう、Emptyの高さ/位置を微調整。配置は [画像の理想像](画像イメージ_自然言語.md) の各室の記述に合わせる。

---

## 5. 雰囲気の照明（館のDNA＝暗い・ティール＆赤）
MMDのMMEに相当する部分。**暗所ベース＋主光1＋赤の差し色1**が基本。
1. **World を暗く**：`World Properties`（地球儀アイコン）Color をほぼ黒（明度0.01前後）。
2. **主光**：`Add > Light > Area` を1つ。部屋の片側から弱め（Eeveeなら Power は距離次第で数十〜数百W、**目で見て“薄暗い”程度**に）。少し寒色（ティール）に寄せると雰囲気が出る。
3. **赤の差し色**：小さな `Point` か `Area` を1つ、赤で弱く。血の記憶の差し色。
4. **レンダラ**：`Render Properties`（背面カメラ型）で **Render Engine = EEVEE**。
5. **色調（シネマ調）**：同 `Render Properties > Color Management > View Transform` を **AgX**（4.x既定）か **Filmic**。Photoshopで後調整するなら **Standard** でもよい。

---

## 6. レンダリングして所定パスに保存
1. **F12**（または `Render > Render Image`）でレンダリング。
2. レンダリング窓で `Image > Save As…`（**Alt+S**）→ **PNG** で保存。
3. 保存先とファイル名をゲームの規約どおりに：
   - `app/assets/images/rooms/r1_north.png`（北）
   - Empty の Z回転を 90/180/270 に変えて `r1_east.png` / `r1_south.png` / `r1_west.png`
4. これで**R1の4枚**が完成。まずは**R1だけ**入れてゲームで確認（パイロット）。

---

## 7. 状態差分（情景変化）＝ライトを変えて撮り直すだけ
出現物は**差分背景に描き込む**運用（[チェックリストB](アセット完全チェックリスト.md#b-背景状態差分-16枚-情景変化に必須)）。3Dなら物理的に切替できる：
| 差分 | やること | 保存名 |
|---|---|---|
| R8 `_dark`（消灯） | 主光をオフ/極小に。東の鏡に人影・赤 | `r8_east_dark.png` ほか4方向 |
| R3 `_lit`（点灯） | 青いブラックライトを足す。北の壁に発光する数字(Emissive板) | `r3_north_lit.png` ほか |
| R10 `_on`（通電） | 北/東のモニタ面を **Emission** で発光、シアン光 | `r10_north_on.png` `r10_east_on.png` ほか |
| R9 `_open`（筒入手後） | 西の戸棚を開いた状態に（扉を回転）、中は空 | `r9_west_open.png` ほか |

- **通常背景側には出現物を描かない**（消えている状態）＝ライトを消した/物を隠したバージョンを撮る。
- 同じシーンを複製（`Scene`複製 or コレクションの表示切替）して撮り分けると管理が楽。

---

## 8. 色を全室で一括統一（最後の仕上げ）
一貫性の総仕上げ。どちらか：
- **Blenderのコンポジット**：`Compositing` タブ → `Use Nodes` → **Color Balance**（影をティール、中間をセピア）や RGB Curves を挟む。全室で同じノード設定を使う。
- **Photoshopでバッチ**：全PNGに同じ調整レイヤー/アクションを適用（既存の色統一運用と同じ考え方）。

---

## 9. 量産の進め方（13室）
1. **R1で1周**して手順に慣れる（ここが山）。
2. **“館の共通シェル”**（壁・床・天井・廊下）を1個作って**全室で使い回し**、家具・小物だけ差し替える＝同じ館感＋工数減。
3. 各室：焦点オブジェクトを置く → カメラ4方向 → 差分があれば撮り増し → 保存。
4. 焦点の位置は [画像の理想像](画像イメージ_自然言語.md)（各室4方向の記述）と、当たり判定の rect（`data/deep_rooms/r*.json`）に合わせると絵とタップ位置がズレない。

---

## 代替ルート：CC0素材で作る（商用が確実）
商用可のMMDステージが見つからない/規約が不安なときは、こちらが安全：
- **Poly Haven**（CC0：HDRI・テクスチャ・モデル）、**ambientCG**（CC0テクスチャ）、**BlenderKit**（無料＋ロイヤリティフリー）、**Fab/Quixel**（無料枠）。
- 手順は 3章以降（サイズ→カメラ→照明→レンダリング→色）が丸ごと共通。**素材の入手元が違うだけ**。
- 箱（壁4枚＋床＋天井）を作り、CC0の家具を置くだけでも成立。

---

## つまずきやすい点（初心者チェック）
- **真っ黒で映らない** → World が暗すぎ or ライト不足。まず主光を強めて位置を確認。
- **カメラが下を向く** → カメラ回転 `X=90°` を忘れている。
- **縦横が横長のまま** → Output の Resolution を 1080×1920 に。
- **材質が真っピンク/白飛び** → MMD材質のまま。まずは薄暗い照明で撮り、必要なら Principled BSDF に変換。
- **重い/落ちる** → Eeveeを使う（Cyclesは重い）。解像度を一時的に下げて構図確認 → 本番だけ1080×1920。
- **PNGがgit管理されない** → 本番画像は `.gitignore` 対象。配布に含めるならコミット対象化 or 配布ビルドへ同梱（[チェックリスト運用](アセット完全チェックリスト.md#命名規約運用)）。

## まとめ（最短ルート）
1. Blender＋MMD Tools を入れる → 2. 商用可ステージを `.pmx` で取り込む → 3. 1080×1920 → 4. 中心にEmpty＋子カメラ、Z回転で4方向 → 5. 暗い照明＋赤 → 6. `r1_*.png` に保存 → **まずR1だけゲームで確認** → 7. 差分・色統一 → 9. 共通シェルで13室展開。
