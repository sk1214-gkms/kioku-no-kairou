# アセット仕様書（画像・イラスト・音）— アムネジィ・ケース / 作話システム版

> 現行の **深い部屋キャンペーン（作話システム）** に必要なアセットを、実データ [data/deep_rooms/](../data/deep_rooms/) から抽出。仕様は [仕様書_v3_作話システム.md](仕様書_v3_作話システム.md) を参照。
> 旧「30部屋ゲーム」のアセット一覧は git 履歴（本書の旧版）に残存。本書は現行版で上書き済み。
> **ファイル名はコードと一致**させること（rect はタップ領域、物は背景に描き込みでOK）。

## 0. 基本仕様

| 項目 | 指定 |
|---|---|
| プレイ領域 | **中央キャンバス 360×640（9:16・縦）**。`DesignCanvas` 上で FittedBox スケール |
| 解像度 | @3x 推奨：背景 1080×1920／ズーム(subview/拡大) 800×800以上／アイコン 256×256 |
| 形式 | 背景＝PNG/JPG（不透明）／オブジェクト・アイコン＝PNG透過 |
| 座標 | `views.<dir>.objects[].rect:[x,y,w,h]` は 360×640座標 |
| **4方向構成** | 各部屋は **東西南北4視点**。原則 **1方向＝1背景**（最大 13×4＝52枚）。簡素な面は省略・流用可 |
| 状態差分 | toggle で見え方が変わる面のみ複数用意（§4の★） |
| HUD | 上部に脳死タイマー（残り時間）＋点灯済みトラウマ文字（黒文字体・§5） |

### エンジンが読み込むパス規約（実装済み・未配置でも安全フォールバック）
- **背景**：`assets/images/rooms/<roomId>_<dir>.png`（例 `r6_north.png`。dir＝`north`/`east`/`south`/`west`）。未配置なら暗色にフォールバック。
- **アイテムアイコン**：`assets/images/items/<itemId>.png`（例 `item_r1_key.png`）。未配置ならアイコン無し（ラベルのみ）。
- 配置したら `pubspec.yaml` の `assets:` に `assets/images/rooms/` と `assets/images/items/` を追加（フォルダに画像を入れてから登録）。
- subview(拡大)・①マーカー・状態差分の差し替えは今後対応（現状は規約外）。

## 1. アートディレクション

- **世界観**：1950〜60年代風ゴシック洋館——だが正体は**瀕死の蓮見鏡介の脳内に再構築された記憶の迷宮**。洋館 × 無機質な医療の二重性。
- **パレット**：暗い青紫〜セピア、**血の赤**がアクセント。**章が進むほど“現実(白い医療)”が滲む**（後半に走査線ノイズ・電極・モニタ）。
- **後半の崩壊**：R10以降は赤い文字化け（VERLUST の予兆）を背景に混ぜると効果的。
- **記憶の正体は嘘**：情景はどこか作り物めいた違和感（不自然な反復、辻褄の合わない陰影）を忍ばせる。
- **統一感**：全背景同一画風・同一光源。AI生成は §8 のプロンプト核を共通化。

## 5. GEDÄCHTNIS / VERLUST 演出（符号化）

- **トラウマ文字は画像ではなく“フォント”で描画**。`fontFamily:'Blackletter'`（**UnifrakturMaguntia** 等の黒文字体・OFL）。配置手順は [assets/fonts/README.md](../assets/fonts/README.md)。旧「回廊文字グリフ18種」の画像セットは**不要**になった。
- HUD：点灯済みのトラウマ文字（T·S·Ä·C·N·I·G·E·D の点灯順）を黒文字体・琥珀色で表示。未点灯は非表示。
- **R13収束（`VerlustRevealScreen`）**：黒背景。断片→アナグラム整列→`GEDÄCHTNIS`（抑圧 **H** が血色で嵌まる）→赤い `VERLUST` が割り込む。背景画像は不要（テキスト演出）。任意で `bg_void.png`（灰色の虚無）を用意すると D 結末と共用可。

## 8. 生成プロンプト集（画像ごと）

各画像の英語プロンプトは **「STYLE_BASE（共通の画風）＋ 各画像の SUBJECT（被写体）」** で組む。英語の方が多くのモデルで忠実。`STYLE_BASE` を必ず前置きして画風を固定する。

### 8-1. 共通（全背景・情景に前置き）
**STYLE_BASE（背景用）**
```
dark vintage 1950s gothic western-mansion interior that is secretly a sterile
medical facility, muted teal-and-sepia palette with a single accent of blood red,
dim cinematic lighting, painterly semi-realistic, oppressive uncanny atmosphere,
faint sense this is a reconstructed false memory, vertical 9:16 composition,
no text, no characters
```
**NEGATIVE（共通）**
```
text, letters, watermark, signature, modern objects, smartphone, bright cheerful
colors, cartoon, anime, lowres, blurry, deformed, extra limbs, people, crowd
```
- 4方向は STYLE_BASE を固定し、SUBJECT の「焦点の家具」だけ差し替えて**同一部屋に見える連続性**を保つ。
- 後半（R10〜R13）は末尾に `faint red digital glitch / data corruption creeping into the scene` を追加。
- 状態差分(★)は `, the room in darkness lit only by [光源], hidden glowing detail revealed` のように暗転版を別生成。

| **Adobe Firefly** | **商用安全**（学習元がライセンス済）＋Photoshop連携で**背景除去/加筆** | 無料枠＋Adobe CC | 可（安心） | 広告収益で権利を気にするなら有力 |
| **Recraft** | **アイコン・透過PNG・スタイル統一セット**が得意 | 無料枠＋有料 | 可 | §8-3 のアイテム8種を揃った画風で量産しやすい |
| **Stable Diffusion（ローカル / ComfyUI・A1111）** | 完全無料・**フル制御・LoRA・透過(rembg)** | 無料（要GPU） | 可（モデル規約注意） | 上級者向け。大量生成・細かい統一に強い |

**おすすめの組み合わせ（実用スタック）**
1. **背景・結末＝Midjourney**（無料で行くなら **Bing(DALL·E3)** or **Leonardo**）で §8 のプロンプトを使用
2. **アイテムアイコン＝Recraft**（透過・画風統一が楽）
3. **透過処理＝**Firefly/Photoshop か `remove.bg`、SD系なら `rembg`
4. **画風の統一＝Leonardo/SD の img2img** で、素材サイト写真や複数AIの出力を STYLE_BASE に寄せる

> 迷ったら：**まず Bing(DALL·E3) 無料**で §8 プロンプトを試し、画風が決まったら **Midjourney か Leonardo** で本生成→**Recraft**でアイコン、が低コスト。
