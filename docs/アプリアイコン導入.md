# アプリアイコン導入手順（自宅PC・turnkey）

**採用案＝V3「頭文字A × 鍵穴＋金の細環」**（Claude Designで作成）。
このゲームの"顔"。以下は、書き出したPNGを Flutter アプリに全サイズ一括生成する手順。**自宅（ネットあり）で実施**。

---

## 0. Claude Design からの書き出し（決定版の出し方）
- **iOS用マスター**：**1024×1024・不透明（透過なし）・角丸なし**の正方形PNG。
  → `assets/icon/app_icon.png` として保存。
- （任意・Android adaptive用）**前景だけ**のPNG：**A＋鍵穴を中央60〜70%のセーフゾーン内**に置き、**背景は透過**。
  → `assets/icon/app_icon_fg.png` として保存。背景色は下の設定で #0E0C14 を使う。

> まず簡単版（`app_icon.png` だけ）で全部作れる。adaptive は後からでOK。

---

## 1. pubspec に追記（自宅で）
`dev_dependencies:` に：
```yaml
dev_dependencies:
  flutter_test:
    sdk: flutter
  flutter_launcher_icons: ^0.14.1   # ← pub getで解決。エラーが出たら最新版に変更
```
トップレベル（`flutter:` と同じ階層）に設定ブロック：
```yaml
# --- 簡単版（まずはこれだけでOK：iOS/Android共通の四角アイコン）---
flutter_launcher_icons:
  image_path: "assets/icon/app_icon.png"   # 1024x1024・不透明
  android: true
  ios: true
  remove_alpha_ios: true                    # iOSは透過禁止→自動で不透明化

# --- 本格版（Android を丸/角丸に最適化したい時だけ、上の android: を消してこちらに）---
#flutter_launcher_icons:
#  image_path: "assets/icon/app_icon.png"
#  ios: true
#  remove_alpha_ios: true
#  android: true
#  adaptive_icon_background: "#0E0C14"              # 館の背景色
#  adaptive_icon_foreground: "assets/icon/app_icon_fg.png"  # 前景(A+鍵穴・透過)
#  min_sdk_android: 23
```

## 2. 生成コマンド（自宅で）
```bash
flutter pub get
dart run flutter_launcher_icons        # 旧: flutter pub run flutter_launcher_icons
```
→ `android/` `ios/` の各サイズのアイコンが自動生成・差し替えられる。

## 3. 確認
- `flutter run` で実機/エミュのホーム画面アイコンが V3 になっているか。
- **48pxでも「A」と「鍵穴」が判別できるか**（＝squint test。潰れるならマスターを見直し）。

---

## つまずき対策
- `flutter_launcher_icons` のバージョンで pub get が失敗 → 数字を最新（pub.devで確認）に変更、or `any` で一旦解決。
- iOSでアイコンが黒枠/透過で弾かれる → `remove_alpha_ios: true` が効いているか、マスターが**不透明**か確認。
- Androidで縁が切れる → 前景を**セーフゾーン(中央60〜70%)**に収める（adaptive版）。
- スプラッシュ画面も作るなら別途 `flutter_native_splash`（今回はアイコンのみ）。

## メモ
- マスターPNG（app_icon.png / _fg.png）は `assets/icon/` に置く。**PNGは配布用に追跡してよい**（プレースホルダ規約と同じ）。
- 予備の顔：案01（Aモノグラム）はローディング/SNS用に取り置き。案V1は好みで差し替え可（同手順）。
