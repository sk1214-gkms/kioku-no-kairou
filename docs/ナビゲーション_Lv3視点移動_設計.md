# Lv3 ナビゲーション設計（視点移動・Myst／森の洋館タイプ）

## 目的
「奥行きのある部屋に立ち、**方向やオブジェクトをタップして視点を移動・接近**して探索する」体験にする。
現状（4方向を"向く"回転＋オブジェクト接写subview）から、**視点ノードを移動する**モデルへ拡張する。

> ⚠ これは全13室の設計・エンジン・アート量に影響する**大きな変更**。**R1で試作→手応え確認→展開**の順で進める（いきなり全室作り替えない）。実機未検証の現状も踏まえ、まずR1を隔離プロトタイプにする。

---

## モデル：視点ノード・グラフ
- **部屋 = 視点ノード(viewpoint)の集合**。プレイヤーは常に「どれか1ノードに立って」いる。
- 各ノードが持つもの：
  - `id`（例 `entrance`, `at_washstand`）
  - `bg`（そのノードから見た背景画像・奥行きのある構図）
  - `objects[]`（ホットスポット。rect＋アクション）
- ホットスポットのアクション（既存＋新設）：
  | アクション | 意味 | 実装 |
  |---|---|---|
  | **`goto: <nodeId>`**（新設） | その方向/対象へ**移動**（視点が変わる） | エンジン追加 |
  | `examine`（＝現行 `subview`） | オブジェクトを**接写**して戻る | 既存 |
  | `reveal` / `lock` / `gives` / `requires_item` / `toggle` … | 調べる・錠・入手・使用等 | 既存のまま |
- **戻る**：移動履歴スタックで1つ前のノードへ（現行の「戻る」ボタンを流用）。＋各ノードに"引く/振り返る"用の `goto` を置いてもよい。

### 「移動」と「接写」の違い（大事）
- **移動(goto)** = 立ち位置が変わり、**新しいホットスポット群**が現れる（さらに奥へ進める）。
- **接写(examine)** = その場で対象を拡大して見る→戻る（立ち位置は変わらない）。
- 両方が組み合わさって「奥へ進む→棚に近づく→扉を開ける→鍵を見つける」の探索になる。

---

## 既存エンジンとの関係（なるべく壊さない）
- 現行の **4方向(N/E/S/W)回転** はそのまま残す＝**部屋の"入口ノード"**として使える（後方互換）。
- 現行の **subview（入れ子）** は **examine（接写）** に流用。
- **追加するのは主に `goto`（ノード間移動）**＝ 現在ノードを履歴に積んで切り替える仕組み。
- データは、部屋JSONに **`viewpoints`** ブロックを持てるよう拡張（従来の `views` を持つ部屋はLv1のまま動く＝段階移行可）。

## データ例：R1（白い部屋）をノードグラフに
```
r1.viewpoints:
  entrance         # 入口に立って部屋を見渡す（奥行きのある広い構図）
    - goto:at_washstand   (北の洗面台の方をタップ→近づく)
    - goto:at_bed         (東のベッドの方をタップ→近づく)
    - goto:at_door        (南の扉の方)
    - goto:at_window      (西の窓の方)
  at_washstand     # 洗面台の前に立った視点
    - examine:mirror / basin
    - cabinet (針金で開く→bg差替→box出現→鍵)   ※現行R1の仕掛けを流用
    - goto:entrance  (振り返って戻る)
  at_bed           # ベッドの前
    - pillow(針金入手) / sheets / underbed
    - goto:entrance
  at_door          # 扉の前
    - door(鍵で脱出)
    - goto:entrance
  at_window        # 窓辺
    - bars / sill
    - goto:entrance
```
- ＝「入口で見渡す→行きたい方向へ進む→その場で調べる→戻る」。今のR1の謎（針金→棚→鍵→扉）は**そのまま載る**。

### 必要アート（R1・Lv3版）
- `r1_entrance`（奥行きのある部屋の見渡し・1枚）
- `r1_at_washstand` / `_at_bed` / `_at_door` / `_at_window`（各立ち位置の絵）
- ＋接写：洗面台の棚(閉/開)、鏡、ベッド接写 等（現行の zoom を流用）
- ＝SDで「立ち位置ごとの視点」を生成（今晩作っている素材の多くは流用可）。

## エンジン変更の要点（R1プロトタイプ）
1. `DeepRoomScreen` に **viewpoint モード**を追加（`room['viewpoints']` があればそちらで動作。無ければ従来の4方向）。
2. 現在ノード `_nodeId` と **履歴スタック** `_navStack` を保持。
3. `_tap` に **`goto` アクション**を追加（履歴に積んで `_nodeId` を切替＝背景・ホットスポット総入れ替え）。
4. 背景 `_bgAsset` は **現在ノードの `bg`**（状態差分 bg_variants も継続）。
5. 「戻る」＝履歴を1つ戻す（空なら部屋の入口ノード）。
6. examine（接写）＝現行 subview をそのまま使用。
7. アイテム/フラグ/錠/branch/脳死タイマー/結末＝**部屋横断の仕組みは不変**（ノード内で発火するだけ）。

> 移動の演出（スライドイン/前進ズーム）は後付け可。まずは切替が動くことを優先。

4. 手応えが良ければ **R2以降へ展開**（1室ずつ）。ダメなら Lv2 に落とす退路も確保
5. 既存の謎・アイテム・結末が壊れないことを各段階で確認（現状すべて未実機検証な点に留意）
