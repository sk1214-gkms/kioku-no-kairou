# アプリ開発仕様書：『記憶の回廊』 v3 ― 作話（Confabulation）システム版

> **位置づけ**：本書が現行実装の正式仕様。[仕様書_v2.md](仕様書_v2.md) は旧「5ステージ/30部屋ゲーム」の仕様で、コード上は legacy（`GameFlow` / `final_room.json` / `endings.json` の `rules`・`thresholds`）として温存。本書が扱うのは **深い部屋キャンペーン（`DeepCampaignFlow`）** 系統。
> 更新日：2026-06-24。実装済みだが **未だ実機/エミュ未検証**（社内＝日本語パスで解析クラッシュのため自宅検証待ち）。

---

## 1. コンセプト（最重要）

本作は「真相を暴く謎解き」ではない。**プレイヤーが解く謎＝主人公の脳が“無罪の自分”を捏造していく作話（confabulation）そのもの**である。

- 主人公 **蓮見鏡介**（白慈館の精神科医）は、恩師・氷室教授を発作的にペーパーナイフで殺害した**真犯人**。
- ゲームの舞台「記憶の回廊」は、その現実から逃れるため、瀕死の蓮見の脳が必死に再構築している**防衛迷宮**。
- 謎を解き、推理を「正解」させるほど、脳は「犯人＝甥」という**美しい嘘を完成**させる。
- 最適化（ノーミス・全逃避）を突き詰めたプレイヤーほど、最も邪悪な結末 **A＋** に至る。逆に、泥臭く**直面**し続けた者だけが、人間性の回復＝**True** に辿り着く。**ゲームシステムそのものが倫理を裁く天秤**。


## 3. ゲームモードと脳死タイマー

全モード共通で **「脳細胞の壊死」カウントダウン**が campaign 全体に1本走る（`DeepCampaignFlow`）。0 で即 **Ending D**。

| モード | 総制限時間 | 備考 |
|---|---|---|
| ノーマル | 1500秒(25分) | 標準 |
| タイマー | 900秒(15分) | 短い |
| ハード | 720秒(12分) | 最短（＋暗号解読等の難化は今後） |

- タイマーは部屋ごとではなく**全体で1本**。各部屋HUD右上に残り時間を表示（残り≤30秒で赤）。
- 残り時間は採点の T（生存度）に直結（→ 第7章）。

## 4. 深い部屋エンジン（`DeepRoomScreen`）

各部屋は **東西南北4視点 × ネスト調査(subview) × アイテム合成 × 多段ロック** の脱出部屋。データは [data/deep_rooms/](../data/deep_rooms/)（`campaign.json` がマニフェスト、`r1.json`〜`r13.json`）。

主なオブジェクト挙動（`views.<dir>.objects[]`）：

| キー | 効果 |
|---|---|
| `reveal` | タップで文章表示 |
| `subview` | ネストした拡大視点へ |
| `toggle` / `states` | 状態トグル（照明など） |
| `requires_state` | 前提状態が揃うまで反応しない |
| `lock`(number/text, `win`) | 暗証錠。`win:true` で脱出 |
| `requires_item` / `on_use_*` | アイテム使用（消費・状態変化・入手） |
| `gives` / `set_flag` | アイテム入手・永続フラグ |
| `combines`(部屋直下) | 2アイテム合成 |
| `branch` | 脱出時に分岐（confront 蓄積）→ 第6章 |

クリア時：`memory_id` を full に、`clear_text`（分岐時は選択肢の `text`）を表示し次室へ。

## 5. 符号化演出：GEDÄCHTNIS → VERLUST

各部屋クリアで、脳に**トラウマ強度順（＝綴り順ではない・不規則）**に文字が点灯。`Tod(死)` `Schuld(罪)` 等と**誤認させる罠**。

| 部屋 | R1 | R2 | R3 | R5 | R6 | R7 | R9 | R10 | R11 |
|---|---|---|---|---|---|---|---|---|---|
| 文字 | T | S | Ä | C | N | I | G | E | D |

- R4/R8/R12（分岐部屋）は**文字なし**。
- **H はどの部屋でも点灯しない**＝脳が符号化を拒んだ「抑圧された記憶」。
- HUD は**点灯済みの文字だけ**を点灯順に黒文字体で表示（未点灯は枠ごと非表示＝単語を伏せる）。
- **R13** で `VerlustRevealScreen` が発火：断片(9文字) → アナグラム収束 → **GEDÄCHTNIS（抑圧 H が血色で嵌まる・全10字）** → **VERLUST** が赤く割り込む。
- 文字描画は `fontFamily: 'Blackletter'`。フォント未配置でも標準にフォールバック（→ 第10章）。

## 6. 分岐（直面／逃避）と confront/evade

R4・R8・R12 で **直面（confront+1）／逃避（evade+1）** を選択し蓄積（フラグ `s_r4_confront` / `s_r4_evade` ほか）。結末の振り分けに使用（扉の明示選択は**置かない**＝創発）。

## 7. 採点：作話完全度（Confabulation Integrity）

```
I = (T × M) × (1 + E / 3)
```

| 記号 | 意味 | 算出 |
|---|---|---|
| T | 生存度(0-100) | `脳死残り / 総時間 × 100`（`survivalT`） |
| M | 作話正解数(0-3) | 30号室で「嘘(lie)」を選んだ数（`meters['confab']`） |
| E | 逃避数(0-3) | `s_r*_evade` の数（`countEvade`） |

実装：[lib/endings_eval.dart](../lib/endings_eval.dart) の `confabIntegrity` / `survivalT`。結末画面の精神鑑定書に T・M・E・I を表示（[verdict_screen.dart](../lib/screens/verdict_screen.dart)）。

## 8. 30号室「確定の儀式」（`FinalJudgmentScreen`）

データ：[data/deep_rooms/judgment.json](../data/deep_rooms/judgment.json)。3問に答え、組み上げた結論を**直視**してから「この記憶を、確定する」で戻れない確定（→ onComplete）。扉の3択は無し。

| 設問 | lie（作話） | truth（真実） | その他 |
|---|---|---|---|
| 真犯人 | 氷室の甥 | 私自身（蓮見） | 館の使用人(decoy) |
| 凶器 | 甥のナイフ | ペーパーナイフ | **銀色のシリンジ(syringe)** |
| あの夜 | 目撃しただけ | 私が手を下した | 記憶にない(decoy) |

確定時に書き込むフラグ／変数：
- `meters['confab']` = lie を選んだ数（M）
- `flags['all_truth']` = 3問すべて truth
- `flags['syringe_chosen']` = 凶器でシリンジ
- `flags['has_culprit_evidence']` = R9で金属製の筒を入手済み（S の前提）

## 9. エンディング（7結末・創発・離散決定木）

判定：[lib/endings_eval.dart](../lib/endings_eval.dart) `evaluateConfabEnding`。**上から評価し最初に一致した1つ**。テキストは [data/endings.json](../data/endings.json) の `confab_endings` → text ID → [data/text_ja.json](../data/text_ja.json)。

| 優先 | 結末 | 条件 | text ID |
|---|---|---|---|
| 1 | **D 精神の死** | 脳死タイマー = 0 | `t_end_D` |
| 2 | **S 深淵の白**〔隠し〕 | 凶器=シリンジ かつ 筒所持 | `t_end_S` |
| 3 | **True 白慈の審判**〔隠し〕 | 全問=真実 かつ 直面×3 | `t_end_true` |
| 4 | **A＋ 完璧な偽物** | 作話正解=3 かつ 逃避×3 | `t_end_A` |
| 5 | **A 作話の綻び** | 作話正解=3（直面あり） | `t_end_A_true` |
| 6 | **B 忘却の揺り籠** | 作話正解<3 かつ 逃避×3（→R1ループ） | `t_end_B` |
| 7 | **C 断罪の重圧** | 作話正解<3（直面あり） | `t_end_A2` |

- **D** はスコア盤・符号を出さず、漆黒に診断書のみの専用表示（`VerdictScreen._deathView`）。
- **B** は `loop_to_stage:1` で回廊の最初(R1)へループ（記憶初期化）。
- 全7本の本文はユーザー確定稿。

## 10. アセット：黒文字体（亜簪文字）フォント

HUD・結末・R13収束の符号を亀甲文字で描くため `Blackletter` フォントを参照。**未配置でも標準にフォールバックしビルドは通る**。有効化は [assets/fonts/README.md](../assets/fonts/README.md)：Google Fonts「UnifrakturMaguntia」(OFL) の .ttf を `assets/fonts/` に置き、[pubspec.yaml](../pubspec.yaml) 末尾の `# fonts:` を解除。

> 数学用フラクトゥール Unicode（𝔗 等）は標準フォントで豆腐(□)になり、Ä にコードポイントが無いため、**通常文字＋黒文字体フォント**方式を採用。
