# 多言語対応（l10n）土台と有効化手順

**方針**：翻訳作業は実プレイでテキストが固まってからだが、**"仕込み"は今から**（[改善ロードマップ §4](改善ロードマップ.md)）。この文書＝そのための土台と、本番での有効化手順。

## いま用意したもの（この会話で作成・現状は未使用＝ビルドに影響なし）
- `lib/l10n/app_ja.arb`／`lib/l10n/app_en.arb`：**UI文言の種**（ボタン・ダイアログ等）。英語版は機械翻訳の下書き＝**ネイティブ監修必須**。
- `l10n.yaml`：gen-l10n の設定（`arb-dir: lib/l10n`／出力クラス `L`）。
- ※`flutter: generate: true` を pubspec に入れていないので、**今はまだ生成もリンクもされない**（安全）。

## 本番での有効化手順（自宅・ネットあり）
1. `pubspec.yaml` に追記：
   ```yaml
   dependencies:
     flutter_localizations:
       sdk: flutter
     intl: any
   flutter:
     generate: true
   ```
2. `flutter pub get` → `flutter gen-l10n`（`.dart_tool` に `L` クラスが生成される）。
3. `MaterialApp` に配線（[lib/main.dart](../lib/main.dart)）：
   ```dart
   localizationsDelegates: L.localizationsDelegates,
   supportedLocales: L.supportedLocales,
   ```
4. Dart内の直書き文言を `L.of(context)!.xxx` に順次置換（下の対象表）。

## 恒久規則（今から守る／守らないと後で全やり直し）
1. **本番アートに"読ませる文字"を焼き込まない**（血文字・メモ・数字はエンジンでテキスト重畳。記号○△□はOK）。→ [3D制作マニュアル](3D制作マニュアル_初心者向け.md) と [画像イメージ](画像イメージ_自然言語.md) に明記済。
2. **新規の文言はデータ（JSON）か .arb に置き、Dart直書きを増やさない**。

## 物語テキスト（結末・部屋の描写）のロケール分割（翻訳フェーズ）
UI文言(.arb)とは別に、**分量の多い物語テキストはデータ側**で分ける：
- 結末本文：`data/text_ja.json` → `data/text_en.json`（[content_repository.dart](../lib/content_repository.dart) の読込パスをロケールで切替）。
- 部屋の reveal/clear_text 等：`data/deep_rooms/ja/r*.json` ＋ `en/r*.json`（ローダはパス切替のみ・エンジン改修最小）。
- **構造ズレ検出**：翻訳版が原版と同じキー/オブジェクト構造かを突き合わせる Python チェックを用意（`tools/` に追加予定）。数字・座標・フラグ名は共通、文字列だけ差し替える。

## 翻訳の進め方（テキスト確定後）
- 分量：UI＋物語で推定 1,500〜2,500 文字列。
- 順序：**英語(EN) → 繁体字中文**。文体が命なので機械翻訳＋**ネイティブ監修**。
- 謎の根幹は言語非依存（HELP/MEMORY/数字錠/○△□）で翻訳を生き延びる＝有利。
