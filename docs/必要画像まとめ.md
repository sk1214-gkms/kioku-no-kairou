# 必要な画像 まとめ（現行メカニクス反映版）

本作で実際に必要になる画像の一覧・枚数・命名・注意点。詳細プロンプトは [アセット仕様書 §8](アセット仕様書.md#8-生成プロンプト集画像ごと)、生成手順は [Firefly](AdobeFirefly_生成手順.md) / [Stable Diffusion](StableDiffusion_ローカル生成手順.md)。
**今は全て一貫プレースホルダで代替済み**（`python tools/gen_placeholders.py` / `gen_item_icons.py`）。本番は**同じパスに上書き**すれば差し替わる。

> 画風統一の要：**基準1枚をスタイル参照に固定→全部生成→Photoshopで色を一括統一**（[Firefly §6](AdobeFirefly_生成手順.md)）。

---

## 1. 背景（通常）── 52枚 ★必須
- パス：`assets/images/rooms/<id>_<dir>.png`（id=r1〜r13、dir=north/east/south/west）
- 比率：縦長 9:16（例 1080×1920）。
- 13室 × 4方向 ＝ **52枚**。各方向の“焦点の家具”を差し替え、同一部屋に見える連続性を保つ。

## 2. 背景（状態差分）── 現状12枚 ★情景変化に必須
ギミック（照明/通電など）で**背景が差し替わる**室。命名は**基準名＋サフィックス**：`<id>_<dir>_<suffix>.png`。
| 室 | サフィックス | 状態（トリガ） | 差分で描くもの |
|---|---|---|---|
| R3 暗室 | `_lit` | ブラックライトON | 明るくなり、**現像壁(数字)が浮かぶ** |
| R8 鏡の間 | `_dark` | 照明OFF | 暗くなり、**鏡に返り血の自分**が映る |
| R10 監視室 | `_on` | 主電源ON | 通電し、**モニターが点灯**する |
- 現状 3室 × 4方向 ＝ **12枚**。今後 R4 等の演出追加で増える可能性。
- ★重要：**「出現するオブジェクトは、この差分背景に描き込む」**（本作は状態が満たされると対象が出現する＝`show_when`。例：R10の`_on`背景には点いたモニターを、R8の`_dark`背景には返り血の鏡を描く）。通常背景側にはそれらを描かない（消えている状態）。
- ✅ **プレースホルダも既に出現物を描画済み**（`gen_placeholders.py` の `DEPICT`）：R8_dark＝返り血の鏡／R10_on＝点いたモニタ／R3_lit＝現像壁。**本番アートはこの"型"どおり、差分背景に出現物を描いて同じパスへ上書き**すればよい（ホットスポットが物に重なる配置になっている）。

## 3. アイテムアイコン（透過PNG）── 15種 ★必須
- パス：`assets/images/items/<itemId>.png`。正方・**背景透過**。
- 一覧：`item_r1_key`（鍵）／`item_frag_a`『破れた紙片(左)』／`item_frag_b`(右)／`item_frag_c`(担当医欄)／`item_chart_half`(復元しかけカルテ)／`item_pass`(通行証)／`item_pass_valid`(受理印つき通行証)／`item_stamp`(受付印)／`item_syringe_empty`(空シリンジ)／`item_needle`(注射針)／`item_vial`(薬液)／`item_syringe_loaded`(針付きシリンジ)／`item_syringe_full`(薬液入りシリンジ)／`item_culprit_evidence`(金属製の筒)／`item_knob`(ドアノブ)。

## 4. 結末キービジュアル（任意）── 最大7枚
- 7結末（A+ / A / B / C / S / True / D）の締め絵。無くても文章で成立。過激表現(流血/自傷)は Firefly で弾かれるため暗示 or Photoshop加筆（[Firefly §5](AdobeFirefly_生成手順.md)）。

## 5. タイトル/UI（任意）
- タイトル背景（1枚）。グリフ（GEDÄCHTNIS/VERLUST）は**フォント**で代替（画像不要。UnifrakturMaguntia を配置）。

## 合計の目安
| 種別 | 枚数 | 必須度 |
|---|---|---|
| 背景（通常） | 52 | ★必須 |
| 背景（状態差分） | 12（現状） | ★情景変化に必須 |
| アイテムアイコン | 15 | ★必須 |
| 結末キービジュアル | 〜7 | 任意 |
| タイトル | 1 | 任意 |
> **最優先で用意すべきは「背景52＋状態差分12＋アイテム15」**。まずここが揃えば"画像あり"で完成度が大きく上がる。

## 命名規約（まとめ）
- 背景：`assets/images/rooms/<id>_<dir>.png` ／ 状態差分：`..._<suffix>.png`
- アイテム：`assets/images/items/<itemId>.png`（透過）
- 未配置でも `errorBuilder` で暗色にフォールバック（クラッシュしない）。
- **PNGは .gitignore で非追跡**（プレースホルダは各PC/CIで再生成）。本番画像を配布に含めるなら、その画像はコミット対象にするか配布ビルドに同梱する運用へ切替。
