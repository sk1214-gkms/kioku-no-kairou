## 1. 「30号室」＝欠落の伏線化（案b）✅実施済み

13室しかないゲームに「30号室」が出る違和感を、**作話の崩壊の証拠**に転用する。
[judgment.json](../data/deep_rooms/judgment.json) の `prompt` に一行を追加済み：

> この回廊には、本当は三十の部屋があるはずだった。だが、崩れゆく脳には——もう十三までしか、保てなかった。

- 表示名「30号室 玄関ホール（最後の審判）」は**維持**。
- タイトル『教授の**不完全な**安楽』と響き合う（作話すら完成しなかった）。

## 2. リプレイ設計（A1＋A3）📋実装待ち

### A1 追想モード（本命・工数中）
**狙い**：結末を変える変数は「選択」だけで謎解きは毎回同じ、という構造を利用し、**決断点だけを連続再生する短縮周回**（1周3〜5分）で7結末回収を可能にする。テーマ整合＝死にゆく脳が「作話をやり直す」行為そのもの。

仕様：
- **解放条件**：本編を1回でもクリア（結末コレクション≧1）。タイトルに「追想 ―作話のやり直し―」ボタン。
- **流れ**：①記憶（R3あの夜／R6犯人／R7凶器：直視or書き換え）→ 分岐（R4／R8／R12：直面or逃避）→ R9金属筒の有無（**本編で一度でも入手済みなら選択式**、未入手なら追想では選べない＝追跡は本編で）→ 30号室の審判3問 → 結末。
- **実装**：既存の editable_memory／branch／judgment 部品を再利用する軽量フロー1本（新screen）。錠・探索・部屋描画はスキップ。
- **制約**：D結末は対象外（タイマーは本編の時間制限モード専用）。スコア/共有は本編のみ、追想は**結末コレクション解放のみ**有効。ヒント不要＝広告なし。

### A3 結末条件ヒント（工数小）
初回クリア後、コレクションの未取得チップ「？？？」に**条件のティーザー**を表示：
- A+「すべてを偽り、すべてから目を背けたなら——」
- A 「嘘は完成した。だが、目は逸らしきれなかった」
- B 「嘘は綻び、それでも背け続けたなら——」
- C 「嘘は綻び、直視だけが残ったなら——」
- S 「標本室の奥に隠された"それ"が、鍵になる」
- True「すべての真実を直視し、すべてに向き合ったなら——」
- D 「刻限の中で、脳が灼き切れたら——（時間制限モードのみ）」

## 3. 収益設計（ハイブリッド）🔎採用候補

**考え方：客を2種類に分け、それぞれに合う払い方を1つずつ。** 本命の収入はIAP、広告は無料層の入り口（動画1本≒2〜3円 vs 買い切り¥480＝広告150回分超）。

| 客層 | 払い方 | 内容 |
|---|---|---|
| 無料カジュアル層 | リワード広告（現行維持） | 進捗連動ヒント1段＝動画1本 |
| ファン・ハード層 | **プレミアム買い切り ¥480〜730（SKUは1個のみ）** | 全広告除去＋ヒント広告なし開放。「没入を守るために払う」商品 |

- **インタースティシャル配置規則**：章の切れ目（R4→R5、R9→R10）と結果画面前のみ。**1周最大2〜3回**。部屋の途中・審判前・結末テキスト前は禁止。
- （任意）クリア後の「開発者を支援」一回きり課金。
- 実装：`in_app_purchase` 追加＋購入フラグ1個＋`ad_service` 連動。広告本実装（AdMob）と同時期でよい。

## 4. 多言語対応（方針）📋

**結論：技術的に可能。翻訳作業は後、ただし"仕込み"の規則は今から有効。**

有利な条件：文字解答が既に英語（R5『HELP』のA=1、R13『MEMORY』）＝謎の根幹が翻訳を生き延びる。数字錠・ダイヤル・手順は言語非依存。テキストは全てJSONでエンジンと分離済み。

**今から有効の規則（違反すると後で全部やり直しになる）**：
1. **本番アートに文字を焼き込まない**。血文字・メモ・数字等の"読ませる文字"はエンジン側でテキスト重畳（記号○△□はOK）。Blender/生成アート制作時の必須ルール。
2. 新規の文言はデータ（JSON）か将来の .arb に置き、Dartへの直書きを増やさない。

**将来の作業（実プレイテスト後・テキスト確定後）**：
- 部屋データのロケール分割：`data/deep_rooms/ja/…`＋`en/…`（ローダはパス切替のみ）。構造ズレは「文字列以外の構造一致チェック」ツールで検出。
- UI直書き文言を Flutter gen_l10n（.arb）へ。共有テキスト・ストア文言もロケール別。
- 分量：推定1,500〜2,500文字列。機械翻訳＋**ネイティブ監修**（文体が命）。順序は EN → 繁体字中文。
