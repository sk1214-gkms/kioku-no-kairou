# 構想：謎・推理の質を上げる設計案（アムネジィ・ケース）

本書は「謎解き推理ゲームとしての質」を一段引き上げるための**設計提案**。現行実装（[仕様書_v3](仕様書_v3_作話システム.md)／`DeepRoomScreen`／`data/deep_rooms/`／`evaluateConfabEnding`）を前提に、データ構造・エンジン変更・スコア連携・段階実装まで具体化する。

## 設計思想：テーマを“遊びの動詞”にする
本作の核は **「謎を解く＝自分に都合のいい嘘（作話）を組み立てる」**。
現状この核は**物語**には効いているが、**操作（メカニクス）**は普通の脱出ゲーム（数字錠・消去法・合成）に留まる。
→ **テーマを操作レベルに引き下ろす**のが質向上の最短路。署名級の柱が次の2つ。

---

# ① 記憶の上書き（Confabulation を“動詞”にする）

## 体験
ある手がかりや推理が **自分（蓮見）を指し示す（＝真実）** とき、プレイヤーは二択を迫られる：
- **正直に解く**：手間が多い／難しい。だが**真実の記憶を保持**＝True/Sルートを残す。
- **記憶を上書きする**：その場で「甥が犯人」版に書き換え、**楽に突破**できる。だが**作話完全度↑**＝A＋（幸福な怪物）へ堕ちる。**真実の記憶は永久に失われる。**

> プレイヤーの「最短で解きたいゲーマー本能」が、そのまま蓮見の堕落になる。テーマの完全な操作化。

## データ構造（room JSON / オブジェクトに追加）
```jsonc
{
  "id": "list", "label": "容疑者表",
  "reveal": "消去法の結論……残るのは、私だ。",          // 真実（デフォルト表示）
  "editable_memory": {
    "fact_id": "culprit",                                // 30号室のどの推理に対応するか
    "truth": { "reveal": "残るのは私だ。直面するなら別の手順で解け", "answer": "蓮見" },
    "overwrite": { "reveal": "そうだ、犯人は甥だ。", "opens": true }, // 書き換えで錠/先へ開通
    "honest_solution": "別の数字錠 or 連結を要求（任意・難）"
  }
}
```

## エンジン変更（`DeepRoomScreen`）
- `editable_memory` を持つオブジェクトをタップ → **真実を見せた上で**ダイアログ：
  - 「**この記憶を書き換える（上書き）**」→ overwrite を適用（`opens:true` なら謎をバイパス）、`meters['overwrite']++`、`flags['ow_<fact_id>']=true`
  - 「**直視して解く**」→ honest_solution へ（無ければ通常解）、`flags['kept_<fact_id>']=true`、`confront` 微増
- 既存の `_clear`/`branch` と同じく `GameState` に記録。

## スコア・結末連携（既存ツリーを壊さない拡張）
- 作話完全度 `I=(T×M)(1+E/3)` の **E を拡張**：`E_total = countEvade(gs) + meters['overwrite']`（上限3でclamp）。書き換えるほど“嘘の完成度”が上がる。
- **30号室の真実選択をゲート**：`flags['ow_<fact_id>']==true`（書き換え済み）の事実は、30号室で**真実の選択肢が ■■（記憶喪失）になり選べない**。
  → **全部書き換えた人は True/S に到達できない**（嘘ルート確定）。**正直に解いた人だけが自己告発（True）できる。**ルートが“最終クイズ”でなく**道中の誠実さ**で決まる。
- `evaluateConfabEnding` は M/E_total/allTruth(=全fact直視＋真実選択) を見るだけで、既存ツリーのまま自然に機能。

## 段階実装
1. **最小**：1部屋（例 R6 記録室）に `editable_memory` を1つ＋上書き/直視ダイアログ＋`meters['overwrite']`。
2. R4/R8/R12 の分岐を `editable_memory` 型へ統合（confront/evade と二重化を解消）。
3. 30号室の真実ゲート（■■表示）＋ E_total 反映。

# ② 証拠連結推理（30号室を本格推理に）

## 体験
今の「犯人/凶器/真相の3択クイズ」を卒業し、**各部屋で集めた“記憶の断片（証拠）”を盤面で連結**して推理を**自分で組む**（『Return of the Obra Dinn』『逆転裁判』『AI:ソムニウム』系）。

## データ構造
**部屋側：証拠の入手**
```jsonc
// オブジェクトに付与（取得で証拠化）
"evidence": {
  "id": "ev_footage", "label": "03:45の防犯映像",
  "reads": { "lie": "甥がナイフを持つ", "truth": "影は私の輪郭だ" }  // 同じ証拠の二面性
}
```
**30号室：主張と必要証拠（judgment.json を拡張）**
```jsonc
"claims": [
  { "id":"culprit", "q":"真犯人", "options":[
     { "tag":"lie",   "summary":"甥",   "needs":["ev_footage","ev_glove"] },
     { "tag":"truth", "summary":"私",   "needs":["ev_bloodyhands","ev_paperknife"], "requires_kept":true },
     { "tag":"syringe","summary":"——",  "needs":["ev_tube"] }            // 隠しS
  ]},
  { "id":"weapon", ... }, { "id":"truth", ... }
]
```

## UI（30号室・連結盤）
- 上段：3つの**主張スロット**（犯人/凶器/真相）。各主張に答え候補。
- 下段：**収集済み証拠チップ**（インベントリ風）。
- 候補を選ぶ → 必要証拠スロットが点灯 → チップを**ドラッグ接続**。
  - 必要証拠が揃う＝**「確定」**（緑）。矛盾する証拠＝**「不安定」**（赤・揺れる）。
- 「審判を下す」で確定内容から結末判定。**シリンジ証拠連結→S**、**自己告発連結（要・記憶保持）→True**。

## 二面性（①と連動するミステリーの肝）
- 各証拠に `reads.lie / reads.truth` の**二つの解釈**。プレイヤーが「どう読むか」を選ぶ＝**道徳の選択**。
- ①で**書き換えた証拠は `truth` 読みが消える**＝嘘でしか連結できない。**①②が噛み合う。**

## 段階実装
1. **Phase 1（軽）**：今の3択を維持しつつ、各回答に「**支える証拠を1枚選ぶ**」を必須化（チップ選択のみ、ドラッグ無し）。証拠未所持なら選べない＝探索が推理に直結。
2. **Phase 2（本命）**：ドラッグ連結盤＋確定/不安定表示＋二面性。

# ①②の相互作用（ここが本作の独自性）
```
道中：記憶を「書き換える」ほど → 真実の証拠が消える＋作話完全度↑
30号室：残った証拠でしか推理を組めない
  ・全部書き換えた  → 甥を断罪する“完璧な嘘” → A＋（幸福な怪物）
  ・正直に解いた    → 自分を指す証拠が残る → True（自己告発）/ S（隠し）も可能
```
**結末は最終クイズの一発勝負でなく、13部屋を通した“誠実さの総和”で決まる。** テーマ・操作・推理・結末が一本の線で繋がる。

# 既存システムとの整合
- **作話完全度 I=(T×M)(1+E/3)**：①の上書きを `E` に合算するだけで自然に取り込める（式は不変）。
- **7結末ツリー（`evaluateConfabEnding`）**：M（作話正解数）・E_total・allTruth・syringe・brainDead の入力が変わるだけで**ロジックは流用**。真実ゲート（①）が allTruth の前提条件になる。
- **ヒント/スキップ・セーブ/ポーズ・ハード**：いずれも上記に直交（影響小）。ハードは「上書きの誘惑が強い（正直解がより難）」と相性◎。

# まず作るなら（最小の縦切り）
**Phase 1：R6「記録室」に①記憶の上書きを実装**
- R6は消去法で「残るのは丙（＝甥に対応）」だが、ここを「**消去法の結論が“私（蓮見）”を指す**」に作り替え：
  - 直視 → 真実を受け入れ別手順で解錠（`flags['kept_culprit']`、confront+）
  - 上書き → 「犯人は甥」に書き換えて即解錠（`meters['overwrite']++`、`flags['ow_culprit']`）
- 30号室で `ow_culprit` なら真犯人＝私（真実）が ■■ で選べない。
- これ1部屋で「①の手触り・スコア・結末への波及」が体験でき、横展開の雛形になる。

→ 合意できれば、この**Phase 1を実装**（データ＋エンジン＋スコア連携）から着手します。
