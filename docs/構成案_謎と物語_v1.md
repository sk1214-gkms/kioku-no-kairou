# 構成案：謎と物語の統合デザイン v1（たたき台）

> ねらい：謎・物語・記憶システム・3モード・分岐・エンディングを **1本の線** に通す。
> 「面白い謎の集まり」ではなく「**1つの大きな謎としての1作**」にする。
> 設計原則は escape/puzzle 設計の定石に準拠（公平・種まき→回収・本物のアハ・物語一体化・温度フィードバック）。

---

## 0. 一行コンセプト
**「これは脱出ではない。あなたが“あの夜”を思い出していく物語だ。」**

## 1. 中核の再定義：『館＝あなたの記憶』
30の部屋は物理的な洋館ではなく、**血まみれで目覚めた“あなた”の、砕けた記憶**。
ドアを開けるたび、抑圧した記憶が1つ戻る。

これで全要素に必然性が生まれる：
- **謎を解く＝記憶を1つ取り戻す**（クリア報酬＝記憶、は既存のまま）
- **タイマーの虫食い＝思い出せない（記憶が壊れたまま先へ）** → 終盤の理解が欠ける
- **ループEnding B＝“あの夜”から抜け出せず再生し続ける**
- **最終室「私は何者だ」＝アイデンティティの再構成**

→ そして **中盤でこの事実自体が“どんでん返し”として明かされる**（後述の見せ場③）。これが「仕組みの理解が一段深まる本物のアハ」。

## 2. 統一メカニクス：記憶帳 × 虫食い × 呼び戻し（本作の看板）
- 各部屋クリアで **記憶シャード（手がかり断片）** が **記憶帳（メニューでいつでも閲覧）** に貯まる。
- **後の部屋の鍵は、前の部屋のシャードを要求**する（例：20号室の鍵に 6・12・18号室の数字が要る）。
  → 定石の「**先に種まき → 後で回収**」を仕組み化。プレイヤーが自分で繋ぐ快感。
- **タイマーで取り逃した部屋のシャードは ■■（虫食い）** で記憶帳に残る。
  → それを要求する終盤の謎が**意図的に解けない／賭けになる** → **悪い結末へ繋がる**。
  → 「記憶の欠落」が雰囲気でなく **機構的な帰結** になる（3モードの存在意義が激変）。
- **ハード**：シャードが回廊文字で記録される（記憶帳も解読が要る）＋偽シャード（decoy）混入。

## 3. 三幕構成（1〜30）と難易度カーブ
各幕＝「導入（やさしく・新要素を教える）→ 盛り上げ → ボス」の波を繰り返す（定石の3フェーズ）。

| 幕 | 部屋 | 体感 | 役割 |
|---|---|---|---|
| **第1幕：館** | 1–10 | 現実の洋館だと思っている | 操作習得→2手→ボス。**種まき**（招待状・日記・血の手・主の肖像・反復モチーフ） |
| **第2幕：綻び** | 11–20 | 辻褄が合わなくなる | 既出の物が“変質”して再登場。**呼び戻し**が要る謎。**中盤どんでん返し**（見せ場③） |
| **第3幕：真実** | 21–29 | 記憶だと知った上で | **矛盾／推理**で「本物の記憶」と「植えられた偽記憶」を選別 |
| **終章** | 30 | 私は何者か | 推理＋3択 → エンディング |

> 反復モチーフ（例：オルゴールの旋律／特定の数字／チェス）を全幕に通し、回収していく。

## 4. 分岐軸『逃避 ⇄ 直面』と 5/15/25 号室
5の倍数の部屋は**複数回答＝選択**（branch機構）。選択は**1本の軸**に効く：

- **5号室**：サイレンが鳴る。証拠を**隠す(逃避)** か **調べる(直面)** か。
- **15号室**：どんでん返し直後。「これは記憶だ」を**否定(逃避)** か **受け入れる(直面)** か。
- **25号室**：真犯人の正体に**目を背ける(逃避)** か **暴く(直面)** か。

→ 各選択でフラグが立ち、**逃避/直面のバランス**が蓄積。これが終盤の結末を左右（中盤の選択が物語を変える、を全編で実現）。

## 5. 看板の見せ場（各幕の節目に“本物のアハ”を1つ）
1. **第1幕中：自分の痕跡** … 血に濡れた自分の手を調べると、部分的な手形が鍵と一致。「私は関与している」初の不穏。
2. **第1幕ボス(10)：記憶の再構成** … 破れた写真をドラッグで組むと、**主と笑い合う自分**。被害者＝見知らぬ他人、が覆る。
3. **第2幕(≈15)：もう一つの顔（どんでん返し）** … 明かりを消すと、主の肖像が **“あなたの顔”** に。**「館は私の記憶だ」** の大アハ。全体が再解釈される。
4. **第2幕ボス(20)：記憶の呼び戻し** … 鍵に過去3部屋のシャードが必要。虫食いだと完成できず賭けに → 帰結。
5. **第3幕(25)：矛盾を暴く** … 自分の記憶3つが食い違う。**植えられた偽記憶**を1つ見抜く。推理の山場。
6. **終章(30)：私は何者か** … 無傷のシャード＋逃避/直面の軸で最終推理 → 結末。

## 6. エンディングへの接続（既存の枠を拡張）
結末 = **memory_score（どれだけ真実を取り戻したか）× 逃避/直面（どう在ろうとしたか）× 遺留品（隠し）× 最終推理**。
- 直面寄り＋高score＋推理正解 → **A＋ 真・無実の証明**（あなたは嵌められた目撃者）
- 逃避寄り＋虫食い多 → **B 終わらない逃亡（ループ）**
- 遺留品所持＋特定経路 → **C 共犯者の告発**
- 中間は A/A′/B′/C′ に配分（既存 endings.json を軸＋分岐フラグで拡張）

## 7. 3モードが生む“3つの体験”
- **ノーマル**：全記憶 full。真相に最短で迫れる。じっくり型。
- **ハード**：回廊文字＋decoy＋呼び戻しの解読負荷。手応え重視。**記憶は full** なので真相到達可。
- **タイマー**：取り逃し＝虫食い。**断片的な理解のまま進む焦燥** → 偽記憶に騙されやすく、Bへ落ちやすい。**同じ部屋が最も“怖い”体験**になる。

## 8. 設計のガードレール（必ず守る）
- **情報を欠かさない**（曖昧さ＝事故）。解後に逆算できること。
- **種まき→回収**を各幕で必ず一回。
- **温度フィードバック**：誤答に「桁数は合う」「2つは正しい」等。N回ミスでヒント解放を促す。
- **アハの後に“ご褒美の余白”**（連続で新難題をぶつけない）。

## 10. プロト第一歩（提案）
1. **branch機構＋逃避/直面フラグ** を実装し、**5号室を“選択分岐”に**（最小で“選択が物語を変える”を体感）。
2. 続けて **記憶帳UI（呼び戻しの土台）** を実装し、**1つだけ呼び戻し謎**を仕込む（看板の検証）。
3. **見せ場③「もう一つの顔」** を既存エンジンで1部屋作り、どんでん返しの威力を確認。

→ この3つが回れば、本構成の“背骨”が動いて見える。あとは部屋を量産。
