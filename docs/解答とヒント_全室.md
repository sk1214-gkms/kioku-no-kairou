# 解答と進捗連動ヒント 全室（解答キー・現行データ準拠）

開発・テスト用の答え合わせ。**各部屋のJSON実データ（`data/deep_rooms/r*.json`）に一致**させた最新版。
各室：**解き方／答え／収束する手がかり／囮・罠／①・分岐／Hard差／進捗連動ヒント**。ネタバレ注意。

> 凡例：①＝記憶の上書き（直視/書き換え）。分岐＝confront蓄積。Hard差＝ハード(hard/hard_t)のみの追加要素。
> **収束型**＝複数の手がかりを突き合わせて初めて答えが出る（各手がかりは「手がかり(N)」パネルに記録される）。

## A. 進捗連動ヒントの仕組み（ユーザーの状態で出し分け）
コードは [deep_room_screen.dart](../lib/screens/deep_room_screen.dart) の `_showHint` / `_hintDone` / `_condMet` / `_maxHint`。

- ヒントは各室 **5段**。各段は「文章のみ」か「文章＋`done_when`（達成条件）」を持つ。
- **押すと、いま“まだ達成していない最初の段”が出る。** 既に終えた手順（`done_when`成立）は**広告なしで自動スキップ**され、飛ばされる＝「今の詰まり」に対応する段だけが提示される（脱出ゲーム定番の進捗連動）。
- `done_when` は **state（装置の状態）／items（所持品）／flags（達成フラグ）** で判定（`show_when`等と共用の `_condMet`）。
- **最終段（答え直結）はストーリーモードのみ開放**。ノーマル/ハードは `_maxHint = 段数−1` で最終段を伏せ、自力導出させる（詰み防止は「……諦めて先へ進む」スキップで担保）。
- 実際に開いた段数だけ `_hintsViewed`（＝広告視聴回数）として結果画面に反映。ボタン表記は「ヒント（広告）」。

**表記**：各段末尾の ［ ］ は「この条件が満たされると自動で飛ばす（＝もう出ない）」トリガ。
［段階開示］＝進捗判定なし（純粋に押すたび深くなる）。［最終/物語のみ］＝答え直結、ストーリー限定。
