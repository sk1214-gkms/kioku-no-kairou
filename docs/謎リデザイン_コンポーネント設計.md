# 謎のリデザイン：コンポーネント式＋収束(メタ)型 設計提案

目的：各謎を**再利用できる"ギミック・コンポーネント"**で組み、**複数の手がかりが収束して初めて解ける**深い謎にする。脱出/謎解きの定石（攻略・設計記事）を踏まえた再設計案。**※本書は提案。合意後に段階実装する。**

---

## 0. 研究まとめ（攻略・設計の定石）
- **構造（部屋全体の組み方）**
  - 線形(Linear)：A→B→C と順に解く。今の本作はほぼこれ。
  - **収束/ファネル(Funnel)**：複数の独立した小謎が**1つの錠/答え**へ合流。←本作に最も足りない深み。
  - **メタ謎(Meta)**：各小謎の"答え"を集めて最終謎を解く。
  - **物体合成メタ**：複数謎の産物（物）を組み合わせて新たな"閃き"を作る（＝本作の合成の発展）。
  - **大謎(ō-nazo)**：最終で、それまでの全体・部屋構造・既出ルールを"読み替えて"解く капーン。R13を発展可。
- **謎の種類（＝コンポーネントの素）**：暗号(シーザー/換字/モールス/書物)、隠し物(隠し戸棚・UV/光で出現)、**相互参照(散らばった値を集めて並べる/記号↔値の対応表)**、多層リビール(段階的に出る)、赤ニシン(囮)、なぞなぞ、物理/順序、音、色/光。
- **原則**
  - **一貫性＝信頼**：規則がブレるとプレイヤーは全部を疑い、ヒント連打になる。同じ表現は同じ意味で。
  - **すべての要素に意味（intentionality）**：エレガントな謎は無駄がない＝"使われない要素"を減らす。
  - **赤ニシンは必ず"戻し"とセット**：囮には必ず否定/誘導の手がかりを併置（行き止まりにしない）。
  - **暗号には必ず解読の足場**（対応表/参照）を室内に置く。
  - **相互参照は"並べ順"の論理**（発見順/大小/左右/時刻）を用意。

## 1. 現状の課題（本作）
- 1部屋＝ほぼ**1つの仕掛けを1直線**で解く（reveal が答えをほぼ提示→錠に入れる）。"複数手がかりの収束"が無い。
- ギミックは**1つのStateクラス内のtype分岐**で実装＝拡張・再利用が利きにくい（dial/sequence/scrub/dialogue/chase…が各々メソッド）。
- 状態(`_states`)は**論理(反応可否)と文章**しか変えず、**情景(背景/出現)は不変**＝ギミックの手触りが薄い。

## 2. 提案：ギミック・コンポーネント体系（再利用部品）
JSONで宣言し、エンジンが解釈する**共通部品**に再編。各部品は《入力(前提)→出力(値/物/状態)》を持つ。

| 部品 | 役割 | 入力→出力 | 既存→拡張 |
|---|---|---|---|
| **clue（手がかり）** | 値/記号/事実を1つ与える | 前提状態/暗号→`memo[key]=value` | reveal を"値を記録する"形に |
| **convergence_lock（収束錠）** | 複数clueの値を集めて開く | memoのN個→解錠 | number/text を**多源(funnel)化** |
| **cipher（暗号）** | 暗号文＋室内に解読表 | 暗号clue＋decoder→平文値 | 回廊文字を正式部品に |
| **scene_state（情景連動）** | 状態で背景/出現を変える | state→bg差分・objの表示/隠蔽・重ね | **新規**（ギミック→情景変化） |
| **combine_meta（物体合成）** | 物の組合せで鍵/閃き | items→item/flag | 既存combineを一般化 |
| **chain（連鎖）** | 解錠が次を解放 | flag/state→解放 | requires_state/grants_flags |
| **red_herring（囮）** | 囮＋必ず"戻し"の手がかり | —（誤誘導） | decoyに redirect を必須化 |
| **capstone（大謎）** | 全体を読み替える最終 | 既出ルール/構造→最終解 | R13を発展 |

### 中核：`memo`（部屋内の"判明した手がかり"置き場）
②証拠連結盤と同じ発想を**各部屋に**。clueを調べると `memo` に値が溜まり、`convergence_lock` はmemoの**複数値の集合/並び**で判定。→「**複数の手がかりが揃って初めて解ける**」が自然に作れる。

## 3. 収束(funnel)の作り方（深みの核）
**1錠＝4方向の小謎の合流**を基本形に。例（R4書斎・改）：
- 北:肖像 → 記号対応表(decoder)を得る（cipher部品）
- 東:書架 → 医学書の冊数=ある桁（clue）
- 西:カルテ → 記号列＝桁の並び（clue）
- 南:扉/金庫 → **収束錠**：北の表で西の記号列を解き、東の冊数で真贋判定 → 4桁が"3方向の合流"で確定
→ どれか1つでは解けない＝**ようやく解ける**手触り。赤ニシン（偽の対応表）には"戻し"(冊数で真贋)を必ず併置。

## 4. 各部屋リデザイン方針（要点）
> 物語・答えの骨子は維持し、"1直線→収束"に作り替える。各室「2〜3の小手がかり＋情景変化＋囮(戻し付き)」を基本に。

- **R1 白い部屋**：導入。収束は浅く。鍵の在処を2手がかり(音＋低位置)で。情景：引き出しを開けると見た目が変わる。
- **R2 準備室**：合成チェーン＋**受付コード収束**（カルテ復元で日付/病棟が判明→扉番号に合流）。
- **R3 暗室**：UVで2つの時刻手がかり→**時刻演算の収束**(止時計＋半刻＋現像)。情景：BL点灯で壁が変化(scene_state)。
- **R4 書斎**：上記§3の3方向収束（対応表＋冊数＋記号列）。
- **R5 診察室**：数列が**3か所に分散**→深さ順で並べ＋シフト＋A=1の多層。囮LIEは"戻し"付き。
- **R6 記録室**：証言×アリバイの**相互参照メタ**（複数証言の矛盾を突き合わせて1人へ収束）＋暗証は別手がかり。
- **R7 廊下**：足跡(規則)×時刻(逆算)の収束。情景：本物の足跡だけ光る(scene_state)。
- **R8 鏡の間**：対峙（各室で直視した記憶の収束＝既にメタ的）。情景：消灯で鏡像が変わる。
- **R9 標本室**：合成メタ＋追跡。隠しS鍵は"3つの気配clue"の収束で在処確定。
- **R10 監視室**：通電→映像。**2モニターの断片を相互参照**して決定的瞬間を割り出す収束。
- **R11 手術室**：手順(メタ)＝各器具のclueを集めて順序を確定。
- **R12 証拠保管室**：全証拠の**時系列メタ**＋捏造除外（既にメタ的）。
- **R13 最後の扉**：**大謎(ō-nazo)**化。各室のグリフ/時刻/真相を"読み替え"て収束（単なるアナグラムから昇格）。

## 5. 段階実装プラン（提案）
1. **エンジン基盤**：`memo` ＋ `convergence_lock`（複数clue合流の汎用錠）＋ `clue`(値登録) を追加。既存錠は内部的にこれへ寄せる。
2. **scene_state**（背景差分・出現/隠蔽）を追加＝情景変化の土台（前回の議論分）。
3. **red_herring**の"戻し"必須化を仕様化。
4. パイロット1室（**R4** 推奨：収束の見本）を作り、手触り確認。
5. 良ければ R3/R5/R6/R10 …へ展開。R13を大謎化（最後）。
- 各段で analyze/test 緑を維持。プレイテストで難度調整。

## 6. 自己評価（辛口）
- **良い点**：収束で「ようやく解けた」の快感が出る／部品化で今後の追加が速い／情景変化で没入が増す／定石（一貫性・戻し付き囮・解読足場）でフェアプレイ担保。
- **注意/リスク**：①**未プレイテストで収束を盛ると理不尽**化しやすい（各clueの発見が前提条件化＝詰まりが連鎖）。→パイロット1室で必ず検証。②`memo`/収束錠は**UI（集めた手がかりの可視化）**が要る（でないと"何が判明済みか"が分からず不親切）。③作業量大（エンジン＋全室データ＋状態差分の絵）。
- **結論**：方向性は強い。ただし**一気に全室は危険**。まず基盤＋R4パイロット→プレイテスト→展開、が安全。
