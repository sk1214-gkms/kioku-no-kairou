# 部屋の背景プロンプト（コピペ用・穴埋め不要）

各部屋の **枠内をまるごとコピー → Geminiの画像生成に貼る → 出た1枚を保存 → コマンド1回**。それだけで4方向そろいます。
（画風・4壁レイアウト・各壁の描写を**全部埋め込み済み**。`[◯の焦点]` を自分で書く必要はありません）

## 使い方（1部屋あたり）
1. 下の「R1」の枠を**まるごとコピー**して、Gemini（Nano Banana＝画像生成）に貼って生成。
2. 出てきた**1枚の画像を保存**（例：`sheet_r1.png`）。
3. ターミナルで：
   ```
   python tools/slice_room_sheet.py sheet_r1.png --room r1
   ```
   → `assets/images/rooms/r1_north.png / r1_east.png / r1_south.png / r1_west.png` が自動生成。
4. パネル間に黒い隙間が残るなら `--gutter 8` を足す。

> 💡 気に入らなければ、Geminiに「もっと暗く」「ティール寄りに」「side wallを消して」と追い足して作り直し、同じ手順で。

> 💰 **無料で詰めて、有料枠で仕上げる**：プロンプト調整は**無料のGeminiアプリ**でOK（下書き扱い＝最終アセットには使わない）。構図・雰囲気が固まったら、**採用する最終版だけ Google AI Studio（従量課金）/ Vertex AI で生成**（商用権利・著作権補償・入力が学習されない＝[配信準備 §4.5](配信準備_ストアとポリシー.md)）。
> - 同じモデルでも**seed固定は無い**ので、引き継ぐのは「**確定プロンプト＋参照画像を添付する手順**」（ピクセル一致はしない）。
> - どうしても同じ絵が欲しい keeper は、その**無料版ドラフトをAI Studioに"参照画像"として上げて**『この構図・画風で』と作らせると近づく（自分の生成物・同じGoogle圏内なので比較的安全）。
> - 調整で見つけた効くフレーズは、この文書の該当プロンプトに**追記して確定版に**しておく。

## ズーム（subview）背景 ― 「深い部屋」用【R1試作】
森の洋館のような"覗き込む探索"を出すため、家具に近づいた**ズーム画面（subview）**にも背景絵が要ります（不可視ホットスポットなので黒画面回避に必須）。
- これは**4分割シートではなく、1枚の close-up 画像**。スライス不要、**保存名そのままで配置**。

### ⚠ 一貫性の鉄則（遠景と接写が食い違わないために）
ズームを別々に作ると、**棚の扉数・鏡の有無**などが遠景とズレて破綻します（R1で発生）。必ず：
1. **スライス済みの"その壁1枚"（例 `r1_north.png`）を添付**（シート全体ではなく単壁）。
2. 「**同じ物・寄るだけ・足し引きするな**」を明示（keep it identical, only move closer, do not add/remove anything）。
3. **パズルに要る物（鏡・箱など）は遠景と接写の両方に**必ず描く。片方だけにしない。
   - 例：接写に鏡を出すなら、遠景の壁にも鏡を足す（遠景を1回編集）。逆に鏡が要らないなら接写から消す。
